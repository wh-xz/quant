# 比赛规则确认清单

策略开发前先确认规则。规则没确认时，所有回测结论都只能算“基于假设的内部实验”。

## 已知信息

- 当前选择：股票场。
- 本机工具：迅投极速交易终端睿智融科版 + XtQuant。
- 本机数据：沪深 A 股日线、常用指数/ETF、部分财务表、板块数据已下载到迅投 `datadir`。

## 待确认规则

| 项目 | 当前状态 | 需要确认的问题 |
| --- | --- | --- |
| 交易池 | 待确认 | 是全 A、沪深 A 股、指定指数成分，还是赛方股票池？是否含北交所？ |
| 基准 | 待确认 | 用沪深 300、中证 500、中证 1000、等权股票池，还是赛方指定基准？ |
| 回测区间 | 待确认 | 训练、验证、排名使用哪些日期？是否滚动评估？ |
| 调仓频率 | 待确认 | 日频、周频、月频、固定 N 日，还是赛方限制频率？ |
| 交易成本 | 待确认 | 佣金、印花税、过户费、滑点和冲击成本如何设置？ |
| 成交规则 | 待确认 | 停牌、涨跌停、一字板、成交量不足如何处理？ |
| 仓位约束 | 待确认 | 总仓位、单票上限、持仓数量、行业上限、现金比例是否有限制？ |
| 数据口径 | 待确认 | 是否允许财务数据？财务公告日如何滞后？价格使用前复权还是后复权？ |
| 提交格式 | 待确认 | 提交 `.py`、`.rzrk`、报告、结果表，还是平台内运行？ |
| 评价指标 | 待确认 | 看收益、超额收益、夏普、回撤、卡玛、稳定性，还是综合排名？ |

## Codex 工作约束

- 没确认的规则必须写成“待确认假设”。
- 策略参数要集中配置，不能把规则假设散落在代码里。
- 回测报告必须写明交易池、基准、成本、调仓频率、回测区间和评价指标。
- 不能只展示最优参数；需要展示稳健性和失败样本。
- 模拟盘/实盘版本必须与回测版本分开。

## 默认临时假设

在赛方规则未补全前，可以用以下假设做内部 baseline，但报告和提交前必须复核：

- 交易池：沪深 A 股，剔除 ST、停牌、新股、低流动性股票。
- 基准：中证 500 或中证 1000，待赛方确认。
- 调仓：周频或 10 个交易日。
- 成本：至少包含佣金、印花税和保守滑点。
- 持仓：分散持仓，限制单票和行业集中。

这些只是开发假设，不是赛方规则。
