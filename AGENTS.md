# Codex Project Rules

本仓库用于中信量化大赛股票场。任何 Codex/代理在本仓库内工作时，必须先遵守本文件。

## 第一优先级：比赛规则先行

在设计、修改、回测、评价或提交任何策略前，必须先确认或显式记录以下规则：

- 比赛赛道：股票场、ETF 场、期货/期权场，或其他赛道。
- 交易池：全 A、沪深 A 股、指定指数成分、赛方股票池、是否含北交所、是否排除 ST/退市整理/新股。
- 基准：沪深 300、中证 500、中证 1000、等权股票池、绝对收益，或赛方指定基准。
- 回测区间：训练区间、验证区间、最终排名区间、是否滚动评估。
- 调仓频率：日频、周频、月频、固定 N 日、事件触发。
- 成本模型：佣金、印花税、过户费、滑点、冲击成本、最小交易单位。
- 成交约束：停牌、涨跌停、一字板、开盘/收盘成交、成交量限制、是否允许未成交顺延。
- 仓位约束：总仓位、现金比例、单票上限、行业上限、持仓数量、换手限制。
- 数据口径：价格复权方式、财务数据公告日滞后、行业分类来源、指数成分历史。
- 提交格式：策略文件格式、报告格式、回测结果格式、是否需要模拟盘运行。
- 评价指标：收益、超额收益、最大回撤、夏普、卡玛、波动率、换手、稳定性、排名规则。

如果规则来源不明确，必须在输出中标注为“待确认假设”，不得把假设当成赛方事实。

## 禁止事项

- 不得凭空发明比赛规则、收益、排名、回测指标或赛方要求。
- 不得在未说明假设的情况下固定交易池、基准、手续费、滑点或回测区间。
- 不得因为本地数据可用，就默认比赛允许使用该数据。
- 不得把回测版策略直接当作模拟盘/实盘版本。
- 不得自动执行真实下单、模拟盘提交或比赛最终提交，除非用户明确要求并确认账户、模式、风险和下一步动作。

## 默认工作方式

- 规则未完整确认时，可以先写 baseline、数据检查、研究 notebook 和回测框架，但必须把规则相关参数集中配置并标注假设。
- 对策略表现的描述必须来自迅投回测结果页、日志、已保存结果文件或可复现脚本输出。
- 策略版本必须与回测结果对应记录，优先使用 `strategies\`、`results\`、`reports\`。
- 发布策略到迅投目录时，优先使用 `scripts\publish_strategy.py` dry-run，再由用户确认后执行。
- `links\` 只作为本机迅投目录入口，不提交真实软连接目标、数据缓存、日志、账号或安装包。
